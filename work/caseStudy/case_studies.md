# FlyRank ML Internship – Baseline Action Scoring Workflow

## The Problem

The dataset contained many content pages with search performance signals, making manual review inefficient. The goal was to create a simple and explainable workflow that prioritized which pages should be reviewed first.

## What I Did and Decided

I explored the dataset and built a threshold-based scoring workflow using search metrics such as impressions, click-through rate (CTR), and average position. I chose a rule-based approach instead of a machine learning model so the scoring remained transparent and easy to explain. I also added reason codes such as `LOW_CTR_HIGH_IMPRESSIONS` and `LOW_CTR_TOP_POSITION` to explain why each page was prioritized.

## What Came of It

The workflow generated an explainable review queue that ranked pages using measurable search signals. It served as a baseline solution that can be extended with more advanced approaches in the future.

### What I Would Do Differently

Next time, I would compare the baseline against a machine learning model and evaluate the ranking using quantitative metrics.
